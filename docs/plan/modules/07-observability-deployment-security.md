# 模块 07：可观测性、部署与安全基线

## 1. 可观测性

### 1.1 指标（Metrics）

- Hub：API QPS、发布成功率、审批耗时、WebSocket 连接数。
- Agent：长轮询挂起数、唤醒次数、KV 写入延迟、缓存命中率。
- SDK：初始化耗时、轮询重试次数、配置回调耗时。

### 1.2 日志（Logging）

统一字段：`trace_id`、`request_id`、`cluster`、`app`、`path`、`version`、`error_code`。

### 1.3 追踪（Tracing）

关键链路：

- 配置发布（Hub -> Agent）
- 热更新通知（Agent -> SDK）
- KV 写入传播（SDK -> Agent -> Redis/DB）

## 2. 部署策略

- Hub：Deployment，多副本无状态。
- Agent：StatefulSet，多副本有状态。
- Redis：高可用部署（哨兵或托管版）。
- 本地 DB：可用嵌入式/Sidecar/独立实例（按环境选择）。

## 3. 安全基线（当前阶段）

- 以内网隔离为主。
- API 预留 Token/mTLS 扩展点。
- 配置敏感字段强制密文存储与传输。

## 4. 故障演练建议

- 演练 1：Hub 全挂，验证 Agent 自治。
- 演练 2：Redis 不可用，验证 L1+L3 降级。
- 演练 3：Leader 故障切换，验证长轮询连续性。
- 演练 4：SDK 初始化失败，验证业务退出与告警。
