# 模块 05：发布编排与审批流程

## 1. 模块目标

将配置变更从“编辑态”推进到“已发布态”，包含审批、分阶段发布、可回滚。

## 2. 流程模型

### 2.1 普通环境

`draft -> pending -> approved -> published`

### 2.2 生产环境

`draft -> pending -> approved -> gray -> prod -> stable -> dr`

每个阶段都支持：审批、发布、回滚。

## 3. 状态机实现建议

- 用显式状态迁移表限制非法跳转。
- 每次迁移记录操作人与时间。
- 失败可重试并保留幂等 key。

## 4. 飞书审批集成

1. Hub 发起审批单。
2. 轮询审批结果。
3. 审批通过触发下一状态。
4. 超时/拒绝进入失败态并通知。

## 5. 模块任务（Tasks）

- [ ] R01：定义 Release、ReleaseStage、ApprovalRecord 数据模型。
- [ ] R02：实现普通环境状态机。
- [ ] R03：实现生产多阶段状态机与阶段回滚。
- [ ] R04：实现飞书审批适配层（创建、查询、重试）。
- [ ] R05：实现发布执行器（按环境向 Agent 推送）。
- [ ] R06：实现回滚执行器（按历史版本恢复）。
- [ ] R07：实现发布审计日志与操作事件流。
- [ ] R08：补齐状态机单测（合法/非法流转全覆盖）。

## 6. 完成定义（DoD）

- 可创建发布单并完成审批后发布。
- 生产阶段可人工逐步推进，且每阶段可独立回滚。
- 审批或发布失败时有明确状态与可恢复操作。
